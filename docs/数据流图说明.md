# EmiyaOJ-Cloud 数据流图说明文档

## 概述

本文档为 EmiyaOJ-Cloud 在线判题系统的完整数据流图（DFD），遵循 Yourdon/DeMarco 结构化分析方法规范，包含以下层级：

| 层级 | 名称 | 描述 |
|------|------|------|
| **Level 0** | 上下文图 (Context Diagram) | 系统作为单一黑盒，展示与外部实体的数据流 |
| **Level 1** | 系统级数据流图 | 分解为 6 个核心处理过程，展示数据存储与跨服务数据流 |
| **Level 2** | 子过程数据流图 | 对 P1 认证授权、P3 代码判题、P2 题目竞赛、P4/P6 博客审核进行深入分解 |

---

## 处理过程清单

### Level 1 过程

| 编号 | 名称 | 对应微服务 | 核心职责 |
|------|------|-----------|---------|
| **P1** | 认证授权 | Gateway + Auth-Service | 登录/登出、JWT 签发与验证、Redis Token 白名单、网关鉴权过滤、RBAC 权限管理 |
| **P2** | 题目/竞赛管理 | Problem-Service | 题目 CRUD、测试用例管理、语言配置、标签管理、题单管理、竞赛管理、报名、排名 |
| **P3** | 代码判题 | Judge-Service | 提交代码、异步判题执行、Go-Judge 沙箱交互、结果汇总计算、提交查询 |
| **P4** | 博客社区 | Blog-Service | 博客/题解/评论 CRUD、点赞/收藏、图片上传(MinIO)、审核任务投递、人工审核 |
| **P5** | AI 聊天 | Chat-Service | 接收用户消息、调用 DeepSeek/OpenAI API、返回 AI 回复 |
| **P6** | 内容审核 | Moderation-Service | 消费 RabbitMQ 审核消息、调用阿里云 ScanText API、回写审核结果 |

### Level 2 子过程

**P1 认证授权子过程：**
- P1.1 登录认证 → Spring Security + BCrypt 密码验证
- P1.2 Token 管理 → JwtUtil 签发/解析 + Redis 白名单存取
- P1.3 网关鉴权过滤 → AuthGlobalFilter (Order=-100)
- P1.4 用户 CRUD → 用户增删改查、状态管理、密码重置
- P1.5 角色权限管理 → 角色/权限 CRUD、RBAC 关联

**P3 代码判题子过程：**
- P3.1 提交创建 → 校验 + INSERT submission (status=0)
- P3.2 异步判题执行 → @Async 异步方法，Feign 拉取题目/用例/语言
- P3.3 Go-Judge 交互 → HTTP POST /run 编译 & 运行
- P3.4 结果汇总计算 → 逐用例比对输出，计算 AC/WA/TLE 等状态
- P3.5 提交查询 → 分页查询 + 逐用例详情

**P4/P6 博客审核子过程：**
- P4.1 博客/题解发布 → INSERT blog (auditStatus=0)
- P4.2 评论互动 → INSERT comment (auditStatus=0)
- P4.3 点赞/收藏 → 维护 blog_like/blog_star 关系表
- P4.4 图片上传 → MinIO S3 API
- P4.5 审核任务投递 → RabbitMQ publish
- P6.1 审核任务消费 → @RabbitListener
- P6.2 阿里云 ScanText → 文本审核 API 调用
- P6.3 审核结果回写 → Feign POST /blog/internal/moderation/result
- P4.6 人工审核 → 管理员 PUT 审核状态

## 数据存储清单

| 编号 | 存储介质 | 内容 | 访问服务 |
|------|---------|------|---------|
| **D1** | MySQL 8.0.31 | emiya_oj_auth, emiya_oj_problem, emiya_oj_judge, emiya_oj_blog 四大数据库 | Auth, Problem, Judge, Blog |
| **D2** | Redis 7 | Token 白名单 (key: `token_{userId}`) | Gateway (ReactiveRedis), Auth (Jedis) |
| **D3** | RabbitMQ 3.13 | 审核消息队列 (Topic Exchange: `emiyaoj.moderation.exchange`) | Blog (发布), Moderation (消费) |
| **D4** | MinIO | 博客图片对象存储 | Blog-Service (S3 API) |
| **D5** | Nacos 2.5.1 | 服务注册与配置中心 | 所有微服务 |

|------|------|------|---------|

## 核心数据流路径

### 1. 用户登录 → 业务请求（完整认证链路）
```
用户 → POST /auth/login → Gateway → Auth-Service 
  → Security验证 → JWT签发 → Redis白名单写入 
  → 返回Token给用户
用户(带Token) → 任意业务请求 → Gateway AuthGlobalFilter 
  → JWT解析 → Redis白名单验证 → 注入X-User-Id/X-User-Name/X-User-Roles 
  → 转发到下游微服务
```

### 2. 代码提交 → 判题结果（核心业务链路）
```
用户 → POST /judge/submit → Gateway → Judge-Service
  → 创建 Submission (Pending) → @Async JudgeExecutor
  → Feign: 获取题目/测试用例/语言配置 → Go-Judge 编译
  → 逐用例: Go-Judge 运行 → 比对预期输出
  → 汇总结果 → UPDATE submission (AC/WA/TLE/...)
用户 → GET /submission/{id} → 轮询获取结果
```

### 3. 博客发布 → 内容审核（异步审核链路）
```
用户 → POST /blog → Gateway → Blog-Service
  → INSERT blog (auditStatus=PENDING)
  → RabbitMQ publish (审核任务)
  → Moderation-Service @RabbitListener 消费
  → 阿里云 ScanText API
  → Feign 回写 Blog-Service: POST /blog/internal/moderation/result
  → UPDATE audit_status (APPROVED/REJECTED/MANUAL_REVIEW)
```
